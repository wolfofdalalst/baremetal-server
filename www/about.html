PROJECT SPECIFICATIONS

BAREMETAL-SERVER IS A PRODUCTION-GRADE HTTP/1.1 SERVER IMPLEMENTATION WRITTEN ENTIRELY IN C. THE PROJECT DEMONSTRATES DEEP UNDERSTANDING OF SYSTEMS PROGRAMMING, NETWORK PROTOCOLS, AND LOW-LEVEL COMPUTER SCIENCE CONCEPTS.

LANGUAGE
C
PROTOCOL
HTTP/1.1
ARCHITECTURE
MODULAR
DEPENDENCIES
NONE
LICENSE
MIT

CORE COMPONENTS

SOCKET LAYER (server.c) ├── Socket creation and binding ├── Connection handling and management ├── Error handling and recovery └── Resource cleanup HTTP PARSER (httprequest.c) ├── Request line parsing (method, URI, version) ├── Header field extraction and validation ├── Body parsing for POST requests └── Protocol compliance checking FILE HANDLER (fileutils.c) ├── Static file serving from www/ directory ├── MIME type detection and assignment ├── File existence validation └── Memory-efficient file reading REQUEST ROUTER (main.c) ├── Request routing and dispatch ├── Response generation and formatting ├── Error page serving (404, 500) └── Connection lifecycle management

DATA STRUCTURES

CUSTOM IMPLEMENTATIONS FOR EDUCATIONAL AND PERFORMANCE PURPOSES:

  • HASH MAP - String-based key-value storage for HTTP headers
  • BINARY SEARCH TREE - Efficient data organization and retrieval
  • LINKED LIST - Dynamic data storage and manipulation
  • QUEUE - FIFO operations for request processing

TECHNICAL DETAILS

SOCKET PROGRAMMING

Berkeley Sockets API for network communication with proper error handling and resource management.

HTTP PROTOCOL

HTTP/1.1 implementation with request parsing, header processing, and response formatting.

MEMORY MANAGEMENT

Manual memory allocation using malloc/free with attention to leaks and buffer safety.

FILE OPERATIONS

Efficient file I/O with MIME type detection and memory-optimized serving.
